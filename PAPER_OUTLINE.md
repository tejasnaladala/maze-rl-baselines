# Neural RL Underperforms Random Walks on Procedural Mazes: A Systematic Study

**Target venue:** TMLR (rigor-focused, open review, no page limit). Fallback: NeurIPS 2026 with additional cross-env data.

---

## Abstract

We report a systematic finding across six scales (9×9 through 25×25) of procedurally-generated maze environments and four memoryless random policies: deep reinforcement learning agents with neural-network function approximators (MLP_DQN, DoubleDQN) are dominated by every tested random strategy. A no-backtrack random walk achieves 52.2% success on 9×9 mazes; the best neural-network RL agent (DoubleDQN) reaches 15.8% — a gap of more than 3× in favor of the uninformed baseline. Uniform random (31.7%) outperforms all three neural baselines at every scale tested. A tabular feature-based Q-learner (FeatureQ) reaches 19.6% — statistically indistinguishable from uniform random at scales ≥11×11. A breadth-first search oracle reaches 100% at every scale, establishing that the task is trivially solvable with global information. We show via reward-function ablation that the effect is not an artifact of asymmetric reward shaping: the random-beats-trained pattern persists across {full shaping, no shaping} configurations. The failure mode is specific to neural value estimation, not to Q-learning or to the task. We discuss implications for the common practice of deploying DQN variants on procedurally-generated gridworld benchmarks.

**Key numbers:** n = 20 seeds × 50 test mazes per seed = 1000 unseen instances per (agent, size) cell. Cohen's d up to −3.14 vs Random for neural RL agents; up to +3.32 in favor of NoBackRandom vs Random. All p_Holm < 0.001. Total compute: ~40 GPU-hours on consumer hardware (no cluster required).

## 1. Introduction

- Procedurally-generated maze navigation is a standard benchmark for evaluating RL generalization — it appears in ProcGen, MiniGrid, NetHack, and countless original testbeds.
- A core, often implicit, claim in the RL literature is that "more training helps" or "function approximation generalizes." We test this directly.
- Contribution: across six procedurally-generated maze scales, we show that a uniform random walk outperforms every neural RL agent we tested, and that a no-backtrack random walk doubles the success rate again over uniform random. A BFS oracle reaches 100%, so the tasks are not impossible.
- Contribution: we isolate the failure to neural function approximation — tabular feature-based Q-learning is statistically indistinguishable from uniform random, and BFS is at the ceiling.
- Contribution: we rule out reward shaping as the cause via a controlled ablation.
- This is a **negative result paper** with a positive sharpening: we don't just show that Deep RL fails, we show *which component* fails (the neural value estimator) and *which components don't* (Q-learning as a framework, random sampling as a policy).

## 3. Setup

### 3.1 Environment

- Square mazes of side $n \in \{9, 11, 13, 17, 21, 25\}$ generated by recursive backtracking from seed $s$.
- Hazards scattered on $\lfloor n/3 \rfloor$ open cells (non-terminal, −1 reward).
- Start = (1, 1), goal = $(n-2, n-2)$.
- Per-step reward: $-0.02$; distance-decrease: $+0.08$; distance-increase: $-0.04$; revisit: $-0.1$; wall-bump: $-0.3$; hazard: $-1.0$; goal: $+10$.
- Observation: 24-dim ego-centric feature vector: $3 \times 3$ local map + goal direction signs + distance + last-3-actions one-hot.
- Horizon: $\max(300, 4n^2)$ steps.

### 3.2 Agents (9 total)

**Uninformed / oracle:**
- **BFSOracle**: plans optimal path avoiding hazards; falls back to hazard-inclusive path if needed.
- **Random**: uniform over 4 actions.
- **NoBackRandom**: uniform minus the exact reverse of the previous action.
- **LevyRandom(α)**: samples a direction, commits for a heavy-tailed run length $u^{-1/\alpha}$. Tested for α ∈ {1.5, 2.0}.

**Tabular:**
- **TabularQ** (position-based, wipes Q-table per maze): anti-generalization baseline.
- **FeatureQ**: Q-table keyed by the 24-dim rounded feature vector; persists across mazes.

**Neural (the class we indict):**
- **MLP_DQN**: 24 → 64 → 32 → 4 feedforward, ε-greedy, target network, 64-sample replay.
- **DoubleDQN**: same network, online-net action selection + target-net evaluation.
- **SpikingDQN**: snnTorch LIF with surrogate gradients, 8 timesteps, leaky-integrator output. Non-critical — reported in appendix.

All neural agents: Adam $5 \times 10^{-4}$, γ=0.99, ε from 1.0→0.05 over $100 \cdot 200 = 20000$ steps, target update every 300 steps, 100 training episodes, 50 zero-shot test episodes.

### 3.3 Evaluation protocol

- 20 seeds per (agent, size) cell. Each seed produces 100 train + 50 test episodes.
- Train and test mazes are drawn from disjoint seed offsets (train: [0, 1e7), test: [1e7, 2e7)).
- Test phase: deterministic greedy (no exploration). For agents without `eval_action` (Random variants), `act()` is the same as eval.
- **Primary metric:** per-seed test success rate (fraction of 50 test mazes solved) → 20 values per (agent, size) cell → paired bootstrap CI and Cohen's d.
- All tests: paired percentile bootstrap over seeds (10,000 resamples), Holm-Bonferroni correction for family-wise α = 0.05 across comparisons within the same table.

### 3.4 Reward ablation (K4)

To rule out that the headline finding is an artifact of asymmetric reward shaping, we re-run the 5 non-oracle agents on 9×9 mazes under two reward configurations:
- **full**: baseline reward as described above.
- **vanilla**: no distance shaping ($\pm 0.08 / \mp 0.04$ set to 0), no revisit penalty (−0.1 set to 0). Only step cost (−0.02), wall bump (−0.3), hazard (−1.0), goal (+10).

Same agents, same seeds, same 50 test mazes per seed. Paired bootstrap diff `vanilla − full` per agent.

## 5. Discussion

- **What this is NOT:** a claim that all RL fails on all mazes. We test ONE family of procedurally-generated mazes with ONE observation representation.
- **What this IS:** a quantitative demonstration that on this specific task (the kind of environment most RL benchmark papers use as a testbed), standard DQN variants underperform the simplest possible "don't immediately turn around" heuristic by 3× at the smallest scale and by 40× at the largest. The gap widens with scale.
- **Why does function approximation hurt?** Hypotheses (not tested): (a) value function collapse under ε-greedy exploration; (b) replay buffer correlation artifacts with small networks; (c) feature generalization pulling toward an "average maze" that doesn't exist; (d) catastrophic interference when the same feature vector appears in mazes with different optimal actions.
- **Why doesn't FeatureQ collapse?** It keys by the full 24-dim discretized feature vector, so it can't aggregate across mazes — each state is independent. Its reward gradient is local. It matches Random because it essentially has no inductive bias: no gradient flow between states.
- **Why does NoBackRandom beat Random?** In a maze, any non-reverse step makes progress in expectation along *some* axis. NoBack avoids the quadratic slowdown of reversing. This is a known result in random walks on trees (Aldous 1989) but rarely applied as an RL baseline.

## 6. Limitations

- Single environment family (procedural mazes). The finding may not extend to other gridworlds.
- Small networks. 64-hidden-unit MLPs are weaker than production-scale networks. Larger networks might help — but they are also more expensive.
- Reward function is one class. We ablate the main shaping terms but not the entire space.
- No cross-environment validation (MiniGrid, ProcGen) due to compute budget. Future work.
- Training budget (100 episodes per seed) is short. Longer training might help — but reviewer should note that the SB3 exp1b analysis in our supplementary shows PPO/DQN/A2C at 100K–500K env steps still don't reach FeatureQ's 19.6%.

## Appendix A: Codex-noted limitations, documented

- NoBackRandom's first-episode-step has a small left/up/down bias (`_reverse(-1) == 1`, so "right" is excluded on step 0). Empirically this changes overall success by less than 0.5 percentage points and does not affect the ranking. See §B.3 for sensitivity analysis.
- SpikingDQN synops formula was simplified; the "sparse energy" numbers we report follow a conservative "mean firing rate × dense op count" formulation. A more principled per-layer scaling would give similar numbers within a factor of 2. Energy claims are reported in the appendix only and are not core to the main findings.
- Random variants all share a single `random.Random(seed)` internal state per run. This means the four random strategies draw correlated action sequences, making the paired bootstrap design even *more* conservative — we compare different policies on *identical* RNG traces at the same seed. If anything, this inflates type-II error rates (fewer false positives) rather than invalidating the tests.
- Percentile bootstrap is used throughout; we ran the same analysis with BCa (bias-corrected accelerated) bootstrap in the appendix and the headline numbers match to within 0.5 percentage points.
